Para qué utilizar moldes:
- Para que las galletas queden iguales
- Para ahorrar tiempo
- El molde se reutiliza
- Rinda el trabajo: Para hacer muchos (a escala - en masa)
- Para no malgastar materiales
- Para que yo pueda tener 100 empleados: las galletas de todos tienen que salir idénticas
- Adaptar el molde para responder mejor a los cambios (e.g.) ponerle nariz a las galletas, en vez de poner mil narices, pongo 1 sola nariz en el molde

A los moldes en POO los llamaron CLASES.
Con un sólo molde puedo hacer muchas galletas, en programación con una sola CLASE puedo tener muchos OBJETOS.

# Conceptos
# 1. Clase: Es el molde (Plantilla, blueprint, modelo) para crear objetos: Producto, Tienda
# 2. Objeto: Instancia de una clase: producto1, producto2, producto3... productoN
# 3. Atributo (propiedad): Una característica que pueden tener los objetos de una misma clase.
# 4. Métodos: Actividades repetitivas
    - Método constructor
    - Rutinas repetitivas


# Pilares
## 1. Abstracción: Tomar lo escencial (estrictamente necesario) a la hora de crear una o muchas clases.
## 2. Encapsulamiento: Proteger los datos de un objeto
## 3. Composición: Cuando un objeto tiene otros objetos por dentro
## 5. Herencia: Cuando un objeto obtiene (hereda) las propiedades de otro objeto base (super). Se utiliza para crear especializaciones (Médico Gastroenterólogo -> Médico General). Se conoce como "ES UN", ejemplo: un médico gastroenterólogo ES UN médico general.
## 6. Polimorfismo: Muchas formas: Lo mismo pero distinto (e.g.). Mutar:
    - Abstracto vs concreto
        - Abstracto: Definición, teoría, no se puede tocar
        - Concreto: No es teoría, existe, es palpable
    - Producto puede actuar/mutar/convertirse como un Producto electrónico, sigue siendo un producto pero con particularidades
    - Producto puede actuar/mutar/convertirse como un Producto alimenticio, sigue siendo un producto pero con particularidades
